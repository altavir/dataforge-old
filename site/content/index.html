Welcome to the DataForge

The Dataforge documentation and this site is currently under construction so please be patient and contact developers if you need information.

Source code could be found at BitBucket repository

Please use Issue tracker to report problems or request enhancements.

For questions no covered by documentation please see our Google group

Main features

Java platform

  • Portable (platform independent, no compilation needed). Runs everywhere one can install JVM.
  • Type-safe.
  • Memory-safe.
  • Many very good scientific libraries.
  • Scripting support with Groovy or Jython.

Meta-data processor

  • Everything is meta-data and meta-data is everything. The analysis process is inferred from meta-data. Meaning the workflow is automatically adjusted for different input data.
  • Declarative description makes all processes parallelizable and state independent.

Context encapsulation

No environmental values and system paths. All external information is stored in the Context.

Modular

Flexible plug-in system instead of monolithic monstrosity.

Does

  • Provide reliable (memory- and type-safe) and flexible platform for data processing automation.
  • Require little programming expertise to construct analysis processes from existing blocks.
  • Have extensible plug-in framework for additional modules.

Does not

Replace data visualization tools like ROOT, JAS or Origin.
